About ClinicalGraph
♢ What this is
ClinicalGraph is a clinical decision support demo that predicts patient disease risk using a Graph Attention Network (GAT). Unlike traditional ML models that treat patients in isolation, this system models 1,688 patients as nodes in a similarity graph and uses graph-based message passing to incorporate the clinical context of similar patients into each prediction.
📈 Datasets
Two publicly available clinical datasets were merged into a unified patient graph:

UCI Heart Disease — 920 patients from 4 hospitals (Cleveland, Hungary, Switzerland, VA Long Beach). Features include age, resting BP, cholesterol, ECG results, chest pain type, and ST depression. Target binarised: 0 = healthy, 1 = any heart disease.

Pima Indians Diabetes — 768 female patients aged 21+. Features include glucose, insulin, BMI, diabetes pedigree function, and pregnancies. Biologically impossible zeros replaced with column medians before merging.
💡 Model architecture
Graph construction: Patient similarity graph built using k-Nearest Neighbours (k=5) with cosine similarity on the 21-feature normalised patient vector. Every patient is guaranteed exactly 5 neighbours — no isolated nodes.

GAT layers: Two-layer Graph Attention Network. Layer 1: GATConv(21→16, 8 heads, concat). Layer 2: GATConv(128→2, 1 head). Trained with Adam, lr=0.005, early stopping on validation F1.

Explainability: Feature attribution via gradient backpropagation. Counterfactual generation via gradient descent in feature space — finds minimum changes to flip prediction.
🔨 Technology stack
Built entirely in Python and JavaScript:
PyTorch 2.0 PyTorch Geometric Flask 3.0 scikit-learn pandas NumPy Vanilla JS SVG animations
⚠ Disclaimer
This tool is for demonstration and research purposes only. It is not a medical device and should not be used to make clinical decisions. All predictions should be interpreted by qualified healthcare professionals. The model was trained on small public benchmark datasets and does not generalise to real-world clinical settings without proper validation.
How it works
1️⃣ Step 1 — Enter patient values
Type any combination of clinical measurements on the Predict Risk page. All fields are optional — any value you leave blank is automatically filled with the population median from the 1,688 training patients. The model handles partial inputs gracefully.
2️⃣ Step 2 — Patient injected into graph
The new patient is added as node #1689 in the patient similarity graph. Five edges are drawn to their nearest neighbours using cosine similarity on the normalised 21-feature vector. This is exactly the same construction used during training.
3️⃣ Step 3 — GAT message passing
The Graph Attention Network runs a forward pass. In Layer 1, each patient aggregates features from their 5 neighbours using 8 learned attention heads — each head learns which neighbours to weight more heavily. In Layer 2, a second convolution aggregates 2-hop neighbourhood context. The attention weights are extracted and displayed in the Similar Patients panel.
4️⃣ Step 4 — Prediction and explanation
The model outputs a disease risk probability. Four explanations are generated simultaneously:

Feature attribution — gradient backpropagation identifies which clinical values contributed most to the score, and in which direction.

Similar patients — the 5 nearest graph neighbours are shown with their real labels, so you can see what the model is comparing this patient to.

GAT attention — which neighbours received the most attention during message passing.

Counterfactual — gradient descent finds the minimum feature changes that would flip the prediction from high risk to healthy, giving a clinically actionable target.
5️⃣ Step 5 — Care plan
The care plan adapts to both the risk level and the specific features that were flagged. If glucose was flagged as a top contributor, the care plan specifically recommends a glucose tolerance test. If ST depression was flagged, it recommends a cardiac stress test. Low-risk patients receive a preventive care roadmap rather than urgent recommendations.
